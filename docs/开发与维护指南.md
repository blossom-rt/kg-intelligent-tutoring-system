# 开发与维护指南

本文档以当前代码为准，面向本地开发、前后端联调和答辩演示维护。最后核对：2026-07-10。

## 1. 工程职责

| 目录 | 职责 |
|---|---|
| `kg-tutoring-frontend/` | Vue 3 单页应用；页面在 `src/views`，接口封装在 `src/api`，路由及角色守卫在 `src/router/index.js`。|
| `kg-tutoring-backend/` | Spring Boot 服务；请求依次经过 `config`、`controller`、`service`、`mapper`，实体位于 `pojo`。|
| `db/` | 建表、演示数据、学习资料补充及旧库迁移脚本。|
| `docs/` | 接口、架构和设计图文档；图源与导出图片位于 `docs/diagrams`、`docs/图表分类`。|

## 2. 请求链路与鉴权

本地开发时，浏览器请求 `/api/kg/**`，Vite 将其改写为后端的 `/api/**`；Docker 部署时由 Nginx 完成同样的转发。后端以 `Result<T>` 作为主要响应包装，前端 Axios 响应拦截器会解包 `data`。

```text
Browser → /api/kg/** → Vite 或 Nginx → /api/** → JwtAuthFilter
        → Controller → Service → Mapper → MySQL
```

除以下接口外，所有 `/api/**` 请求均须携带 `Authorization: Bearer <token>`：

- `POST /api/auth/login`
- `POST /api/auth/register`
- `POST /api/auth/send-code`
- `POST /api/auth/reset-password`

JWT 默认有效期为 24 小时。角色仅用于前端路由守卫和服务端业务逻辑中的当前用户上下文；新增接口时仍应在服务端执行角色与数据归属校验，不能只依赖前端页面限制。

## 3. 本地配置

复制 `kg-tutoring-backend/src/main/resources/application-template.properties` 为同目录的 `application.properties` 后填写：

| 配置项 | 说明 |
|---|---|
| `spring.datasource.*` | 本地 MySQL 连接信息；默认数据库名为 `kg_tutoring_db`。|
| `jwt.secret` / `jwt.expiration` | JWT 签名密钥与有效期（毫秒）。|
| `deepseek.api.*` | AI 接口的 Key、地址和模型；未配置有效 Key 时 AI 请求不会得到正常模型结果。|
| 邮件相关 `spring.mail.*` | 邮箱验证码发送所需配置；请勿提交真实账号或授权码。|

`application.properties` 已被忽略，不应提交密钥。Docker Compose 使用容器内数据库地址 `mysql:3306`，而宿主机访问数据库应使用 `localhost:3307`。

## 4. 运行与验证

```bash
# 数据库（从仓库根目录执行）
mysql -u root -p < db/init_full.sql

# 后端
cd kg-tutoring-backend && ./mvnw test

# 前端（另开终端）
cd kg-tutoring-frontend && npm install && npm run build
```

开发启动改为后端执行 `./mvnw spring-boot:run`、前端执行 `npm run dev`。登录后可分别使用 `student`、`teacher`、`admin` 测试三类工作台；账号信息见项目根 README。

## 5. 数据脚本使用原则

- 新环境优先执行 `db/init_full.sql`，不要手工混用旧脚本。
- 只需恢复演示业务数据时，按 `cleanup.sql → seed_all.sql → enrich_learning_materials.sql` 执行。
- `db/migrate/` 只面向特定历史库结构，执行前先阅读脚本中的适用条件并备份数据库。
- Docker 的初始化脚本只会在数据卷首次创建时运行；已有 `mysql_data` 数据卷不会自动重新导入。

## 6. 已知接口对齐项

以下两个前端封装目前没有对应的后端 Controller 映射；调用会得到 401（未登录时）或 404（登录后），不应在演示流程中依赖：

| 前端调用 | 期望路径 | 建议替代 |
|---|---|---|
| `getUserInfo()` | `GET /api/auth/info` | 登录响应已返回 Token 和角色；资料页使用 `GET /api/user/profile`。|
| `getStudyRecords()` | `GET /api/student/study-records` | 当前已实现学习看板、路径详情和学习记录更新；如需列表，应补 Controller、Service 与前端页面。|

接口新增或修复后，请同步更新 `docs/api.md`，并以 Controller 注解作为最终核对依据。

```bash
git status --short
```

重点检查：不提交 `application.properties`、`.env`、真实 API Key、邮箱授权码、`node_modules`、`target` 或数据库导出文件。
